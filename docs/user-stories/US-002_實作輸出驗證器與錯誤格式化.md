### US-002：實作輸出驗證器與錯誤格式化

**作為** 開發者  
**我想要** 建立 AJV 驗證器與錯誤訊息格式化方法  
**以便** 系統能區分 `SchemaError` 並在 logs 顯示可讀原因

**輸入格式**：
- `unknown` JSON 解析結果
- `lib/schema.ts` 匯出的 schema

**輸出格式**：
- `lib/validator.ts`
  - `validateContentPack(data): { ok: true; data: ContentPack } | { ok: false; errorType: "SchemaError"; message: string }`
  - `formatAjvErrors(errors): string`

**驗收條件**：
- [ ] invalid payload 會回傳 `SchemaError`
- [ ] 錯誤訊息包含具體欄位路徑（如 `blog.outline`）
- [ ] valid payload 可回傳 typed `ContentPack`
- [ ] 不直接在 UI 層操作 AJV 原始錯誤物件

**依賴關係**：
- 需先完成：`US-001_定義核心型別與Schema.md`

**優先級**：P0  
**相關功能對應**：Validate 規則、錯誤顯示
